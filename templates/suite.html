Optional reading assistant (eye-tracking concept)
If you allow the camera, the UI can use gaze or dwell-time heuristics to spot “stuck” words. This demo uses mouse dwell on underlined terms and mirrors a webcam preview only (no model shipped).

Technical execution pipeline
End-to-end flow from clinical authoring to patient comprehension. Flask serves this concept; swap demo endpoints for LLM, ClinicalBERT (medicalai/ClinicalBERT), and DEFT-backed retrieval.
1
Input

Clinician provides the clinical note and chooses complexity (Professional / Standard / Plain).

2
Generation

LLM rewrites the note to the target reading level; DEFT mappings inform substitutions.

3
Verification

ClinicalBERT (or equivalent) scores semantic match and fact preservation before sign-off.

4
Doctor review

Interactive term mapping supports cross-checking; clinician edits and approves.

5
Publishing

Approved simplified text is released to the patient portal alongside the original.

6
Reading

Patient toggles versions, uses DEFT tooltips, and optionally enables dwell- or gaze-based assistance.
